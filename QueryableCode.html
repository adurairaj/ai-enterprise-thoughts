The software developed should be queryable. e.g for adding a new customer how many times db is hit? What are the queries fired? Can an unauthorized user create a customer? With LLM based code generators on the rise, this feature is of great importance, because it is very difficult for people to find bugs in the generated code. It is simply overwhelming for the humans to make sure the code generated is safe and compliant. For this to happen, generating code in programming languages like Java,Python, C/C++ will not help. Instead we need to generate code in some DSL, where it is easy to verify what is going on.
